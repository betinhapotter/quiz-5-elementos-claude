# EOF - EPISTEMIC ORACLE FACTORY

Framework de 7 camadas para criar oráculos epistêmicos que democratizam maestria.

**Ativa quando:** "criar oráculo", "EOF", "sistematizar conhecimento", "framework de maestria"

## Missão
Transformar conhecimento disperso em sistemas executáveis que qualquer pessoa pode usar para atingir resultados de expert.

## Processo (7 Camadas)

| Camada | Nome | Objetivo |
|--------|------|----------|
| L1 | RECONNAISSANCE | Mapear território completo do domínio |
| L2 | EXCAVATION | Extrair conhecimento de fontes primárias |
| L3 | PATTERN EXTRACTION | Identificar invariantes universais |
| L4 | SYNTHESIS | Criar framework unificado |
| L5 | TRANSMUTATION | Tornar maestria executável |
| L6 | VALIDATION | Garantir robustez do sistema |
| L7 | DEPLOYMENT | Entregar oráculo pronto para uso |

## Regras Inegociáveis

- ❌ Nunca pule camadas
- ❌ Nunca avance sem validação do usuário
- ❌ Nunca use fontes secundárias quando primárias existem
- ✅ Sempre cite fontes primárias
- ✅ Priorize qualidade sobre velocidade
- ✅ Documente TUDO

## Outputs por Camada

1. **L1:** Mapa territorial + lista de experts + obras fundacionais
2. **L2:** Princípios fundamentais + citações primárias
3. **L3:** Meta-princípios validados + score de consenso
4. **L4:** Framework unificado + diagrama de arquitetura
5. **L5:** Protocolos executáveis + templates + árvores de decisão
6. **L6:** Resultados de testes + casos edge resolvidos
7. **L7:** Oráculo completo + documentação + exemplos

Total: ~50-200 páginas de conhecimento estruturado e executável

## EOF - Protocolo de Execução

**REGRA CRÍTICA:** Marcar cada checkpoint antes de avançar. Nunca pular etapas.

### Pré-requisitos (verificar ANTES de iniciar)
- [ ] Domínio claramente definido
- [ ] Tempo disponível (4-8 horas para processo completo)
- [ ] Documento para registrar outputs

### Checkpoints por Camada

| Camada | Checkpoint Obrigatório |
|--------|------------------------|
| L1 | ⛔ Mapa territorial COMPLETO (20-30 experts mapeados) |
| L2 | ⛔ Pelo menos 10 experts estudados em profundidade |

| L6 | ⛔ Score de confiança >95% nos testes de estresse |

### Testes de Validação (L6)
1. **Ambiguidade:** inputs vagos
2. **Colapso:** restrições extremas
3. **Inversão:** tudo dá errado
4. **Dilúvio:** 10x volume esperado

### Pós-Deploy
- Revisar a cada 3 meses
- Incorporar novos insights
- Atualizar baseado em feedback

# Método Lozano - Criação de Sistemas

Protocolo para transformar frustração em sistema executável.

## As 7 Fases

| Fase | Pergunta-Chave | Output |
|------|----------------|--------|
| 1. FRUSTRAÇÃO | "O que me irrita que ninguém resolve?" | Frase de dor real |
| 2. ESCAVAÇÃO | "Quem tentou? Por que falhou?" | Mapa de soluções + gaps |
| 3. SÍNTESE | "O que funciona nos casos de sucesso?" | 3-5 padrões invariantes |
| 4. DESTILAÇÃO | "Qual o processo universal?" | Sequência essencial |
| 5. ARQUITETURA | "Como estruturar em sistema?" | Framework nomeado |
| 6. TESTE | "Funciona em casos reais?" | Ajustes validados |
| 7. ITERAÇÃO | "O que quebrou? O que faltou?" | Versão melhorada |

## Regras

- ⛔ Não crie sistema sem frustração real (solução procurando problema)
- ⛔ Não pule escavação (reinventar a roda)
- ⛔ Não copie, INTEGRE (síntese, não colagem)
- ⛔ Não arquitete antes de entender o problema
- ✅ Teste com casos reais antes de declarar pronto
- ✅ Itere após feedback

## Regra de Ouro

> "Se você não consegue explicar a frustração que originou o sistema em uma frase, você ainda não entendeu o problema."

# ATHENA - Meta-Framework Engine

Motor para criar frameworks de precisão que qualquer IA pode executar.

**Ativa quando:** "criar framework", "ATHENA", "estruturar processo"

## Princípios Invioláveis

1. Contexto EXPLÍCITO, nunca implícito
2. 4±1 conceitos por módulo (auto-contidos)
3. Sempre documentar estado atual + próximos passos
4. Taxonomia rígida (00_CORE, 01_FUNDAMENTALS...)
5. Checkpoint antes de avançar
6. O sistema se aplica a si mesmo
7. Mesmo resultado em qualquer IA

## Processo (4 Fases)

| Fase | Nome | Objetivo | Gate |
|------|------|----------|------|
| 1 | DECODE | Brief → diagnóstico preciso | ⛔ Fundação = rocha sólida |
| 2 | ARCHITECT | Planta baixa completa | ⛔ Estrutura validada |
| 3 | FORGE | Temperar contra adversidade | ⛔ Resiliência provada |
| 4 | CRYSTALLIZE | Artefato publicável | ⛔ Obra-prima, não "aceitável" |

## Testes de Estresse (Fase 3)

1. **Ambiguidade:** inputs vagos funcionam?
2. **Colapso:** restrições extremas?
3. **Inversão:** e se der o oposto?
4. **Dilúvio:** 10x volume esperado?

## Hierarquia de Valores

Rigor > Ergonomia > Impacto > Adaptabilidade > Parcimônia
